UTSA Reg+ Privacy Policy

Effective May 2026 · v1.0 · Built by a UTSA student, for UTSA students

Short version

  • We collect nothing about you.
  • There is no server we operate. All data stays in your browser.
  • Your saved schedule lives in Chrome storage — same tier as a bookmark.
  • Network requests go only to public UTSA, RateMyProfessors, and Simple Syllabus endpoints.
  • The source code is public. Every claim here is verifiable.

What the extension stores locally

All data is stored in chrome.storage — your browser, never transmitted to any server we control.

Key What it holds Why
schedule:v1 CRNs of sections you've added or saved So your schedule survives restarts
prefs:v1 Theme (light/dark), F1 mode, GPA-protect mode Your settings
rmp:v1:* RateMyProfessor ratings keyed by instructor name Cache — avoids re-fetching on every popup open
syllabus:orgs:v1 UTSA org tree from Simple Syllabus (6-hour TTL) Cache only
asap:sections:v1 Sections seen on asap.utsa.edu while the extension was active Powers live seat count and section data on cards

chrome.storage.sync holds your theme preference so it follows you across Chrome profiles. No PII, no UTSA student ID, no transcript data, no email, no name.

What network requests the extension makes

All requests go from your browser directly to the endpoints below. Nothing is proxied through any server we operate — we don't have one.

Endpoint When What's sent
ratemyprofessors.com/graphql When you view a section card with an instructor The instructor's name as a search string
asap.utsa.edu Only when you navigate there yourself Your normal browser request — the extension reads the page you already loaded
utsa.simplesyllabus.com Once per 6 hours when a course detail dialog opens Public read-only API, no auth
catalog.utsa.edu At build time only (developer-side scrape baked into the bundle) Public catalog — not fetched at runtime

The extension never sends your saved schedule, CRNs, preferences, or any UTSA-specific data to any third party.

What we do not do

  • No analytics — no Google Analytics, Mixpanel, Amplitude, or anything else.
  • No remote error tracking — no Sentry, no Bugsnag.
  • No advertising or tracking pixels.
  • No A/B testing framework.
  • No telemetry of any kind, anonymous or otherwise.
  • No login, no account, no password. We never ask.
  • No remote logging. Errors stay in your browser console.

Permissions the extension requests

Permission Why
storage Saves your schedule and preferences locally in Chrome
alarms Schedules a 12-hour background job to evict stale RMP cache entries and refresh the Simple Syllabus org tree — keeps data fresh without requiring you to reopen the extension

Host permissions are limited to: asap.utsa.edu, catalog.utsa.edu, utsa.simplesyllabus.com, and www.ratemyprofessors.com. We do not request <all_urls>.
